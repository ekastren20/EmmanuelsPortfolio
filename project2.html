Dealer Management Application

User interface for multiple pages and components of our inhouse, C# application using Blazor and DotNet platforms. This project highlights skills in user interface and front end development.

Five Months

UI & Front End Developer

Four People

HTML / CSS / Blazor / Bootstrap / C#

The Objective

Our current application is web based, and was made a long time ago. Therefore, it is very slow with certain functionalities that require pulling data from a large source and it is outdated on appearance. This has caused a setback in customer satisfaction and overall efficiency. The team and I created a new application using the Blazor platform with server-side functionality to create a slicker and faster application. While the developers focused on the back-end, my job was to set the stage using the Blazor template, and then design the user interfaces of various pages and components as our application took form. After design, I took action with front end development coding. This helped the team decide where certain information and functions would go, so that the back-end code could be implemented. Adjustments took place as the application continued to take form. Challenges included timeline, deciding which page or component was appropriate to create next, and understanding C#, Blazor and DotNet cross functionality.

Blazor Bootstrap Template

Step one required me to create somewhere to start. Luckily, Blazor works in tandem with Bootstrap, making it easy to create a fully functioning starter application. I navigated to https://demos.blazorbootstrap.com/getting-started, and followed the steps to create the starter application you see with the purple ribbon.

Building From The Template

From this template, the team and I created the first two navigations, a list of all the applications (customers), and a tab to create a new application. These are the most important functions. I worked in blazorbootstrap to implement icons in the navbar and apply a gredient. Then, I used front end development to create the applications table with a "view" button.

Step 3: Mind Map

Initial “Mind Map” and User flow from Login. This did not end up being our final User Flow as used in the prototype. One key change was signing up as a new member instead of logging in. The Mind Map helped us figure out the different branches the user should go after the Login and Home Page. Something left out of the prototype but included in this Mind Map is the option to choose International vs. Local Community Boards.

Step 4: User Flow

One of our major challenges was establishing a User Flow. How would someone navigate our app under various circumstances, such as being a first time user vs. someone with a well established account?


To your right you'll see our paths determined with Figma, and below you will see the final prototype windows.

Step 5: User Testing & Iterations

While conducting user tests, and from our own observations, we found that several key changes were necessary. For example, having a clear and distinct payment plan for users, displaying a labeled icon bar, and consistent patterns and visuals that run across all pages.

Conclusion

From going to a crowded and overwhelming information hub to a well organized and crisp looking site, our final prototype accomplished the inital goals set forth. User tests revealed our site to make it more user-friendly with emphasis on organization of material and navigation while making it look legitimate to the thematic material.
