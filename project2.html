Dealer Management Application

User interface for multiple pages and components of our inhouse, C# application using Blazor and DotNet platforms. This project highlights skills in user interface, front end development and understanding Blazor Bootstrap utility.

Five Months

UI & Front End Developer

Four People

HTML / CSS / Blazor / Bootstrap / C#

The Objective

Our current application is web based and outdated. Therefore, it is very slow and it's appearance doesn't look great. This has caused a setback in customer satisfaction and overall efficiency. Our goal was to create a new application using the Blazor platform with server-side functionality to create a slicker and faster application. As the only UI and front end developer, my job was to design the main theme and "blueprint" of the application, mimicking the old one but making iterations along the way. This would create an outline for the back end developers so that they could implement their C# code. Adjustments occurred as the application continued to take form, and many meetings took place to discuss. Challenges included creating a theme, using blazorbootstrap, custom coding and CSS isolation.

Blazor Bootstrap Template

Step one required me to create somewhere to start. Luckily, Blazor works in tandem with Bootstrap, making it easy to create a fully functioning starter application. I navigated to https://demos.blazorbootstrap.com/getting-started, and followed the steps to create the starter application you see with the purple ribbon.

Building From The Template

From this template, I created the first two navigations, a list of all the applications (customers), and a tab to create a new application. These are the most important functions. I worked in blazorbootstrap to implement icons in the navbar and apply a gradient. Then, I used front end development to create the applications table with a "view" button. My teammates implemented the code to pull in the data for a list of applications you see here.

Adding the Theme

I found a dark navy blue color to match the company's color scheme, this would be labeled "main-blue" in the CSS. I applied a blue gradient to the already created gradient, and added the main-blue to buttons, headers and table lines. A light blue background was added to accompany the darker user interface.

Home Page Results

The home page displays the most important information initially: all applications and relevant information. As the developers added data pulling and C# code, I worked alongside with them to create a properly functioning interface. This included adding a filters bar, search bar and pagination. Challenges included creating a color scheme for the different application statuses, and creating a max number of applications to display.

Application Page Results

The second most important page is the "New Application" page, where users can create a new application with neccessary information like address, contact information and employment history.

Main Login

Creating the Main login was simple, once Blazor's heirarchy was understood. The main login was a page that needed to be loaded within the "Main Layout" of the Blazor Application. I was able to figure this out while creating the interface of the main login. At the same time, I created a "logout" button at the bottom of the main side bar. The main login display's the theme with a granular blue background.
